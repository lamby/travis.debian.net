travis.debian.net

Easily test your Debian packages on travis-ci.org

What is this?

travis.debian.net is a hosted script to make it easier and cleaner to build Debian packages on the Travis hosted CI service without duplicating configuration across 100s of repos.

Quick setup

  • Enable builds for your package on travis-ci.org
  • Ensure "Only build where .travis.yml is present" is enabled
  • Add .travis.yml to your git-buildpackage branches: 
    $ wget -O.travis.yml http://travis.debian.net/travis.yml
  • If you are using GitHub, your package will be built automatically after pushing.

How it works

First, travis.debian.net detects which release of Debian the branch is targeting based on its branch name. The experimental suite and all varieties of backports are intelligently detected.

A throwaway Docker image is then created containing the corresponding build-dependencies

The package is then built in a clean environment which has no access to the the internet. If the build fails, this is reported back to Travis.

Lintian is then run on the result and the binary packages and source dir are saved in debian/buildd
